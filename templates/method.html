Data Sources

Data were collected from various sources (see table). Assigning pricing and feature data to the correct locale posed a challenge, due to inconsistent naming conventions across datasets. This was solved using geocoding and a modified text comparison technique inspired by species-to-species ortholog matching in biology (see InParanoid) to correctly assign data to the right locale.

Analysis

Pricing and feature data are stored in a Postgresql database on the application server. Locales within a user-defined search area are retreived and a linear regression performed real-time on the actual pricing and selected features (X) to obtain expected pricing. The percentage differences between actual and expected pricing are then reported to identify over- and under-valued locales.
